# Список переводов для ручной проверки

**Дата:** 2026-01-14
**Модуль:** optics_pos_ru54fz
**Метод перевода:** Яндекс Переводчик

---

## Переводы, требующие проверки

### 1. Circuit Breaker (CB) статусы

**Текущий перевод:**
- `CB: CLOSED` → `CB: закрыт`
- `CB: OPEN` → `CB: открыт`
- `CB: HALF_OPEN` → `CB: полуоткрыт`
- `CB: Unknown` → `CB: неизвестен`

**Возможные альтернативы:**
- Оставить аббревиатуру CB как есть (технический термин)
- Или заменить на полный перевод: "Автоматический выключатель: закрыт/открыт/полуоткрыт"
- Или использовать русскую аббревиатуру: "АВ: закрыт/открыт/полуоткрыт"

**Рекомендация:** Оставить `CB:` как есть (понятно техническим специалистам и операторам касс после обучения).

---

### 2. Статус "Critical"

**Текущий перевод:**
- `Critical` → `Критично`

**Возможная альтернатива:**
- `Critical` → `Критическая ошибка` (более подробно, но длиннее)

**Рекомендация:** Оставить `Критично` (краткое и понятное для UI индикатора).

---

### 3. "Connection Timeout"

**Текущий перевод:**
- `Connection Timeout` → `Превышено время ожидания`

**Возможная альтернатива:**
- `Connection Timeout` → `Тайм-аут соединения` (технический термин)

**Рекомендация:** `Превышено время ожидания` более понятно для неподготовленных пользователей.

---

### 4. "Connection Failed"

**Текущий перевод:**
- `Connection Failed` → `Ошибка соединения`

**Возможная альтернатива:**
- `Connection Failed` → `Соединение не установлено` (более формальное)

**Рекомендация:** Оставить `Ошибка соединения` (краткое и понятное).

---

### 5. "Connection Successful"

**Текущий перевод:**
- `Connection Successful` → `Соединение успешно установлено`

**Возможная альтернатива:**
- `Connection Successful` → `Подключение выполнено` (короче)

**Рекомендация:** Оставить текущий вариант (полный и понятный).

---

### 6. "receipts" (множественное число)

**Текущий перевод:**
- `receipts` → `чеков`

**Контекст:**
- `15 receipts (12%)` → `15 чеков (12%)`

**Проблема:** Склонение по числительным в русском языке:
- 1 чек
- 2-4 чека
- 5-20 чеков

**Рекомендация:** Оставить как есть (для простоты), так как в UI используется только родительный падеж множественного числа.

---

### 7. Длинное сообщение о возврате

**Текущий перевод:**
```
"Возврат заблокирован: исходный чек еще не синхронизирован с ОФД.

Исходный заказ: %(ref)s
ID фискального документа: %(doc_id)s
Статус синхронизации: Ожидание

Дождитесь синхронизации исходного чека перед обработкой возврата.
Проверьте статус буфера оффлайн в индикаторе POS."
```

**Возможные улучшения:**
- "Проверьте статус буфера оффлайн" → "Проверьте статус офлайн-буфера"
- "Дождитесь синхронизации" → "Пожалуйста, дождитесь синхронизации"

**Рекомендация:** Проверить тональность и стиль сообщения (императив vs вежливая просьба).

---

### 8. "POS indicator"

**Текущий перевод:**
- "в индикаторе POS" (не переведено)

**Возможная альтернатива:**
- "в индикаторе точки продаж"
- "в индикаторе кассы"

**Рекомендация:** Заменить на "в индикаторе кассы" (более понятно операторам).

---

## Переводы, оставленные как есть (технические термины)

Следующие термины **не переведены** и оставлены в английском виде:

1. **KKT** (ККТ) — Контрольно-кассовая техника
2. **OFD** (ОФД) — Оператор фискальных данных
3. **URL** — Унифицированный указатель ресурса
4. **POS** (в некоторых местах) — Point of Sale

**Рекомендация:** Эти термины можно заменить на русские аббревиатуры:
- `KKT Adapter` → `Адаптер ККТ` ✅ (уже переведено)
- `OFD` → `ОФД` ✅ (уже переведено)
- `POS` → `Касса` или `Точка продаж` (в зависимости от контекста)
